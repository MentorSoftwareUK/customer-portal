# Mentor Customer Portal & Events System - Project Brief

## Project Overview
Development of a custom customer portal and events management system for Mentor SaaS platform, integrating with existing WordPress public website, HubSpot CRM, and QuickBooks accounting system.

## Technical Stack
- **Frontend:** Vue.js with Tailwind CSS
- **Backend:** Node.js
- **Database:** MongoDB
- **Components: ** Flowbite
- **Hosting:** AWS or existing WHM VPS CentOS server
- **Payment Gateway:** Stripe
- **Integrations:** HubSpot API, QuickBooks API, FileMaker/Better Forms

## Architecture Overviewcd
- **Public Website:** WordPress (existing) - event listings and details pages for SEO
- **Customer Portal:** Subdomain with custom Node/Vue application
- **Integration Layer:** REST APIs and WordPress shortcodes to bridge public site and portal

---

## Core System Components

### 1. Events Management System

#### Event Types
- Webinars
- Lunch and Learns
- Podcasts
- Other custom event types

#### Event Features
**Audience Segmentation:**
- Customer only (Mentor SaaS subscribers)
- Non-customer only (with paid option)
- Both customers and non-customers
- Ability to charge non-customers via Stripe while customers attend free

**Public WordPress Pages:**
- Event listing page with search and filtering
- Individual event detail pages
- SEO-optimized event content
- Registration CTAs that redirect to portal subdomain

**Event Registration Flow:**
- Registration form on portal subdomain
- Collect: Name, Email, Company, Phone, any custom fields
- Payment integration for non-customers (Stripe) when event requires payment
- Email verification/confirmation
- Automatic contact creation/update in HubSpot (do NOT update company name for existing contacts)

**Event Communication:**
- Confirmation email with event details and meeting link (Teams, Riverside, or configurable platform)
- Reminder emails with configurable timing (default: 2 days before event)
- Thank you email (default: 24 hours post-event) with:
  - Event recording (once edited)
  - Presentation slides
  - Any additional documents
- All email timing should be configurable via admin settings (not hardcoded)

**Integration Requirements:**
- Push event registration data to HubSpot for marketing/sales tracking
- Track attendance and engagement metrics
- Store meeting platform links (Teams, Riverside, etc.)

### 2. Authentication & User Management

#### Authentication Flow
**For Potential/New Users:**
1. User enters email on portal login page
2. System checks HubSpot for contact with "live customer" status
3. **If live customer found:**
   - Pre-fill registration form with details from HubSpot
   - Prompt user to set password
   - Option for passwordless login via email passcode/magic link
4. **If not live customer:**
   - Standard registration form
   - Optional: Google or Microsoft OAuth for quick registration
   - Create new contact in HubSpot automatically

**Authentication Methods:**
- Email/password
- Passwordless login (email-based passcode or magic link)
- Google OAuth (for non-customers)
- Microsoft OAuth (for non-customers)

**User Status Definitions:**
- **Mentor Customer:** Active subscriber to Mentor SaaS (status from HubSpot)
- **Non-Customer:** Registered user who has inquired, attended events, or engaged but not subscribed

## Notes for Development Team

1. **HubSpot Company Name:** Never update company name when syncing user data - this is critical to avoid data conflicts
2. **Email Timing:** All event email timing must be configurable via admin settings, not hardcoded
3. **Provision Types:** System must support three types: Supported Accommodation, Children's Home, 18+
4. **Non-Customer Definition:** Non-customers are those who engage but don't subscribe to Mentor SaaS - event registration does not make them customers
5. **OAuth for Non-Customers:** Google/Microsoft login should be available but not required
6. **SEO Priority:** Event pages on WordPress must be SEO-optimized with proper meta tags, structured data, and content
7. **Meeting Platforms:** System should be flexible to support Teams, Riverside, or other platforms via configurable links
